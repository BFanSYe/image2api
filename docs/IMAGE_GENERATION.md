# 图片生成说明

## 分辨率策略

image2api 支持 1K/2K/4K 三类图片请求。高清结果必须来自上游原生能力, 不做伪 2K/4K 放大。

推荐行为:

- 1K: 可走普通兼容图片生成路线.
- 2K/4K: 优先走支持 `gpt-image-2` 的原生 Responses/Image generation 路线.
- 上游不支持目标分辨率时, 返回明确失败原因, 不自动降级为假高清.

## 结果落库与展示

图片生成完成后应满足:

- 任务状态为成功.
- `generation_results` 有结果记录.
- 结果 URL 指向本地缓存或对象存储.
- 浏览器使用 GET 能返回图片内容.
- 前端预览能读取图片自然尺寸.

## 常见问题

### 任务成功但前台看不到图片

检查:

1. 结果 URL 是否为空.
2. URL GET 是否返回 `200`.
3. `content-type` 是否为 `image/png`, `image/jpeg`, `image/webp` 等图片类型.
4. 后端是否只保存了上游 file id, 但没有缓存图片文件.
5. 上游 Responses 事件中图片 base64 是否被正确提取.

### 2K/4K 失败但 1K 成功

优先检查:

1. 当前账号是否是 API key 类型高清账号.
2. 上游是否真实支持 `gpt-image-2`.
3. 上游是否支持请求中的尺寸参数.
4. 后台模型价格和模型映射是否启用 2K/4K.
5. 生成日志和上游日志中的错误码.
